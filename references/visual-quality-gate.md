# Product, Originality, Craft, and Divergence Gate

Evaluate every prototype individually, then evaluate the three directions as a set. A later strength never compensates for an earlier failure.

## 0. Reference containment

First-round exploration must be reference-free unless the user explicitly asks to attach or emulate a source. Mobbin research remains useful, but a source image has disproportionate visual weight in image generation and can turn a strong charter into a literal, familiar product template.

For a later targeted edit, attach an accepted source only to repair one named craft failure. Fail the attachment, not necessarily the selected direction, when the source is generic at the intended scale, depends on product-specific semantics, or makes the result safer, more literal, or more ordinary. Keep the original when that happens.

## 1. Product fidelity

Fail the draft when:

- a functional control, action, data entity, navigation item, or state cannot be traced to the product contract;
- a required action, state, or information is missing or visually subordinated;
- reference-derived navigation, modes, labels, entities, controls, workflows, or brand remnants appear;
- the interface suggests the product behaves differently from the user's requirements;
- placeholder text, lorem ipsum, invented gibberish, or inaccurate critical copy appears.

Do not apply this test to visual craft. Non-functional icons, illustrations, typographic treatments, material, color fields, or graphic motifs are valid when they serve the direction and do not imply a new product capability. Ask of a functional element: “Would the product still need this if no reference had shown it?” Ask of a visual element: “Does this strengthen the chosen direction or explain the screen's job?”

## 2. Originality

Originality constrains product structure, not visual craft. Fail the draft when:

- its layout skeleton, control placement, or component sequence is recognizably copied from one reference;
- it retains source-product chrome, distinctive icons, modes, pagination, feedback rows, or domain content;
- it is effectively a reskin with different copy and brand color;
- a reference attachment dictated the product body instead of only the visual system.

References contribute visual soul, never product body. Deep visual transfer is the goal.

## 3. Craft and visual reasoning

Fail the draft when it has no deliberate visual argument: default template typography, uniformly boxed content, identical component treatment, flat color with no role, or no visible craft inherited from an accepted anchor or the direction charter. A reference-free direction is judged by the charter, not by the absence of an attachment.

Check that the screen uses design fundamentals deliberately:

- **Contrast** creates a clear focal point and priority, not just a dark or light background.
- **Repetition** turns a type, icon, rule, shape, or color motif into a coherent rhythm rather than a pile of unrelated controls.
- **Alignment and proximity** reveal what belongs together and make reading paths feel intentional.
- **Color** distinguishes state, hierarchy, or atmosphere; it is not merely a page fill.
- **Visual assets**—iconography, illustration, signal, texture, or graphic metaphor—appear only when they improve comprehension or make the direction memorable. Their absence is acceptable only when type, composition, and material carry equivalent intent.

Also fail generic or dated AI styling: unrequested oversized controls, excessive rounded containers, heavy shadows, glossy gradients, faux luxury, faux futurism, emoji navigation icons, all-caps micro-headers, colored glow or tinted shadows, decorative stat banners, identical icon-topped feature cards, implausible safe areas, cross-platform legacy patterns, 3D charts, over-segmented pies, or illegible dense labels.

## 4. Pairwise divergence

Lay the three results side by side before delivery. Fail a pair when it shares the same silhouette, reading path, grouping logic, focal device, and component sequence with only a changed palette, theme, material, typeface, or dark mode.

Compare each pair explicitly:

- Can a viewer identify each direction's visual protagonist without reading labels?
- Does each use a different spatial grammar or grouping of the same product capabilities, unless the user locked placement?
- Does each have a different contrast and color logic rather than the same layout recolored?
- Would removing color still leave three different compositions?

If the answer exposes a reskin, regenerate the weaker direction from an opposing charter. Never repair failed exploration by changing only the background.

## Recovery order

1. Remove functional elements that lack product justification.
2. If reference structure leaked in, restate the visual-only scope and regenerate.
3. If a direction is generic, strengthen its visual proposition. Remove a weak attachment before adding more prompt detail; otherwise choose a stronger anchor.
4. If a later attachment weakens a selected direction, discard it and return to the reference-free original.
5. If directions collapsed together, rewrite the weakest charter around a different composition and visual protagonist, then regenerate it.
6. If only one detail fails in a selected, otherwise-valid screen, edit that generated prototype in place.
