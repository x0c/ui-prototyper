# UI Prototype Quality Gate

Inspect every image at full size and thumbnail size. Do not deliver a polite, valid, generic interface.

## Pass all four gates

1. **Truth.** The active state, decisive action, and essential information are obvious. No invented behavior, placeholder text, competitor residue, or inaccurate critical copy.
2. **Hero.** At thumbnail size, a viewer sees one deliberate visual event before they can read labels. It explains or embodies the product moment; it is not a decorative background behind a conventional interface.
3. **Visual grammar.** Contrast establishes a focal point. Repetition creates rhythm. Alignment and proximity make groups intentional. Color has separate jobs. Type, iconography, material, illustration, or graphics have a reason to exist.
4. **Difference.** Each direction has a different visual protagonist, silhouette, reading path, grouping logic, and color behavior. Removing color must not turn them into the same screen.

## Automatic failures

- A rail, status strip, centered card, and controls area appear by habit rather than from a user-fixed layout.
- The screen is a complete application inventory: navigation, settings, and minor tools have consumed the space needed by the current moment.
- The hero could be deleted and the result would still be an ordinary dashboard with different labels.
- All controls receive equal box weight, default rounded containers, or the same visual treatment.
- The direction is distinguishable from another only by palette, dark mode, background texture, or typeface.

## Recovery

Do not cure a generic output with more component instructions. Remove nonessential coverage, make the visual proposition more physical and specific, then regenerate that single direction. If the selected result is structurally right but one detail is wrong, edit the generated prototype in place. If a later reference makes it safer or more literal, discard the attachment.
